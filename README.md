# Управленческий контур — платежи · ТМЦ · контроль

Веб-система управленческого учёта по конфигурационным картам Бориса: платёжные документы и накладные как документы двойного контроля денежных и материальных обязательств.

**Демо:** откройте сайт и нажмите «Демо-данные» — загрузится сценарий со всеми механиками.

## Что реализовано

### 1. Платёжные документы (входящие / исходящие)
- `ID_Deal (UUID)` — связка со сделкой (договором)
- `Date_Payment_Execution` — факт движения денег (зачисление / списание)
- `Date_Material_Execution_Plan` — план перемещения ТМЦ, автоматически считается из юридического блока договора («отгрузить в течение N дней после оплаты»), с возможностью ручной корректировки

### 2. Накладные (приходные / расходные)
- `ID_Deal (UUID)` — связка со сделкой
- `Is_Real (Boolean)` — реальность действия: виртуальные накладные (бумажные корректировки) проводятся, но **не попадают** в управленческие регистры и не порождают плановых дат
- `Date_Material_Execution_Fact` — факт отпуска/приёмки ТМЦ
- `Date_Payment_Execution_Plan` — план оплаты = факт + отсрочка из договора

### 3. Логическая матрица проведения
Кнопка «Провести» исполняет матрицу из спецификации: при `Is_Real == ДА` обновляются график ТМЦ, график CashFlow и матрица ресурсов; при `НЕТ` — управленческие регистры игнорируются. Каждое проведение протоколируется в журнале.

### Регистры и рабочие места
- **График ТМЦ** — таймлайны план/факт перемещения ценностей по сделкам
- **CashFlow-календарь** — кумулятивный денежный поток (факт + план) и календарь выплат/поступлений
- **Матрица ресурсов** — управленческий баланс на 6 колонок: деньги (факт/план), ТМЦ (факт/план), дебиторка, кредиторка
- **Красные флаги** — автоматические сигналы снабженцу, продавцу и бухгалтеру: срыв сроков поставки/отгрузки, просроченная дебиторка, календарь выплат кредиторам

## Технически
Статический сайт без сборки и бэкенда: HTML + CSS + JavaScript, данные в `localStorage` браузера, экспорт/импорт в JSON. Разворачивается на любом статик-хостинге (Vercel, GitHub Pages).

Локальный запуск:

```bash
python3 -m http.server 8737
# открыть http://localhost:8737
```
